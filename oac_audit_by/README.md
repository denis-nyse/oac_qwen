# OAC-Audit-BY
Автоматизированный сканер соответствия требованиям Приказа ОАЦ №66 от 01.03.2025 (Минск, Республика Беларусь).

## Описание
Программный комплекс предназначен для автоматизированного контроля состояния защищенности рабочих станций и серверов под управлением ОС Microsoft Windows на соответствие обязательным требованиям Приказа ОАЦ №66 для выбранных классов информационных систем (ИС).

## Возможности MVP (Версия 0.1.0)
- **Интерактивный выбор классов ИС**: Множественный выбор из списка (4-ин, 4-спец, ..., 3-дсп).
- **Автоматические проверки настроек ОС**:
  - П. 3.5: Минимальная длина пароля (≥ 8 символов).
  - П. 1.1: Включен ли аудит событий входа (Logon).
  - П. 3.7: Тайм-аут блокировки экрана (≤ 15 мин).
  - П. 4.1: Отключена ли учетная запись Guest.
- **Проверка наличия СЗИ**: Поиск файлов/служб популярных в РБ продуктов (Авест, Касперский, Secret Net). *Требуется ручное редактирование конфига*.
- **Двойная верификация**: Использование основных и резервных (fallback) команд для надежности.
- **Отчетность**: Генерация детального HTML-отчета с результатами.
- **Логирование**: Запись хода выполнения в лог-файл для отладки.

## Требования
- **ОС**: Microsoft Windows 10/11, Windows Server 2016+ (MVP поддерживает только Windows).
- **Права**: Запуск от имени Администратора (рекомендуется).
- **ПО**: Python 3.9 или выше.

```
oac_audit_by/
├── main.py                 # Точка входа (CLI)

│   ├── rules.yaml          # Правила проверок (требования ОАЦ)

│   └── report.html.j2      # Шаблон HTML-отчета

```

## Инструкция по запуску
1. Откройте терминал (PowerShell или CMD) **от имени Администратора**.
2. Перейдите в папку проекта:
   ```bash
   cd path\to\oac_audit_by
   ```
3. Запустите скрипт:
   ```bash
   python main.py
   ```
4. Следуйте инструкциям в консоли:
   - Выберите классы ИС (пробел - отметить, Enter - подтвердить).
   - Дождитесь завершения проверок.
5. Откройте сгенерированный отчет из папки `reports/`.

## Настройка проверок СЗИ
Файл `config/szi_templates.yaml` содержит список проверяемого ПО. По умолчанию добавлены примеры для:
- Авест Континент
- Kaspersky Security Center
- Secret Net Studio

**Важно:** Вы должны вручную отредактировать этот файл, указав актуальные пути к исполняемым файлам и названия служб для вашего программного обеспечения, если они отличаются.

Пример добавления нового продукта:
```yaml
- name: "Название ПО"
  description: "Описание"
  paths:
    - "C:\\Path\\To\\File.exe"
  service_name: "ServiceName"
  mandatory_classes: ["3-дсп"]
```

## Логирование
Логи работы сохраняются в папке `logs/` с именем вида `oac_audit_YYYYMMDD_HHMMSS.log`. Используйте их для диагностики ошибок проверок.

## Статус реализации требований
| Требование | Статус | Примечание |
|------------|--------|------------|
| П. 3.5 (Пароли) | ✅ Реализовано | Проверка через `net accounts` |
| П. 1.1 (Аудит) | ✅ Реализовано | Проверка через `auditpol` |
| П. 3.7 (Блокировка) | ✅ Реализовано | Проверка реестра |
| П. 4.1 (Guest) | ✅ Реализовано | Проверка локальных пользователей |
| П. 7.10 (СЗИ) | ⚠️ Частично | Только проверка наличия файлов/служб |
| Криптография (П. 5.x) | ⏳ В планах | Требуется ручная проверка в MVP |
